RMS task class derived from task base class.

Detailed Description
The RMS task receives the acceleration raw data from ADXL data store task, calculate RMS and with the help of class HitDetector hit values. RMS, medium and critical hit values will be send to RMS store task together with a timestamp. The class provides an idle detection timer, and for medium and critical hit detection an instance of class HitDetector. It provides the function update_rms to calculate RMS and hit values and to send them to the RMS store task.

◆ task()

void task ( void  )
virtual

RMS task function.

The function directly enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eAdxl37xSensorData: Member function update_rms will be callled to process the received raw data samples from acceleration sensor. Afterwards function checks if RMS task is in idle state, indicated by flag idle. If RMS task is not in idle state receive count rx_cnt will be incremented by 1, the message released back to the pool and the function returns. If RMS task is in idle state, the message is the first data message after sensor wake up. The flag idle will be set to false and the idle detection timer will be started. The power manager task will be informed by sending message eAdxl37xWake from sender eAdxlRms to power manager mailbox, before incrementing rx_cnt and releasing received message to the pool.
  • eTimerExpired: The pointer pElem will be set to 0, to prevent releasing the message because it is allocated static. The function checks the idle condition, which will be true, if no new data has been received within the last timer period of 1 ms. In this case the flag idle will be set to true, the timer will be stopped and sample task will be informed by sending message eAdxl37xIdle from sender eAdxlRms to eAdxl37xTask mailbox. Finally the function set the previous receive counter rx_cnt_prev to the value of the current receive counter rx_cnt.
  • eFwUpdateActive: For testing purposes only, print out debug message at warning level.

    For more details see also message sequence charts in section Measurement

◆ update_rms()

void update_rms ( PoolElem *  pElem)

Function to update the RMS and hit values and send result to RMS storage task.

Parameters
pElemPointer to the received message

The function extracts the acceleration raw data of type struct adxl37x_sensor_value from the payload of the message and enter a loop over the number of raw data samples. In the loop the x, y and z raw values will be converted to micro meter per squared seconds. These calculated values will be powered by 2 and added to the current square sums x_square_sum, y_square_sum and z_square_sum. Afterwards the calculated z-value will be passed into function inpit of the two hit detector instances. If n_sample reached the maximum of samples_per_sec * n_seconds, the final result will be calculated by dividing the square sum values through the number of samples and extracting the root by calling function sqrt for x, y and z axis. The result are the x, y and z RMS values. The number of medium and critical hits will be retrieved by calling the getter function getNumHits for both hit detector instances.
Afterwards a message of type eAdxl37xRmsData with attached structure of type struct adxl_rms_data with the calculated RMS and hit values and a timestamp will be send to RMS store mailbox by calling function sendMsg provided by Mbox class.
After processing of the loop, the function checks if all samples has been processed and in this case the functions returns. Otherwise square sum and maximum values will be set to 0 and the hit counter will be reset by calling member function init before entering a new loop over the remaining raw data samples where again x, y and z raw values will be converted to micro meter per squared seconds, the maximum in x, y and z direction calculated, also powered by 2 and added to the current square sums x_square_sum, y_square_sum and z_square_sum, before sample counter n_sample will be incremented by 1.
