Detailed Description
The nRF9160 LTE modem integrates RFFE, radio and baseband. It supports worldwide operation, enabling cIoT products without regional specifc variants. It consists of baseband processing and RF parts, which together implement a complete 3GPP LTE release 13 (Rel-13) Cat-M1 and Cat-NB1 and LTE release 14 (Rel-14) Cat- NB1 and Cat-NB2 capable product. Following is a part of the LTE modem:
  • RF transceiver
  • Modem baseband (BB)
  • Embedded flash/RAM
  • Modem host processor and peripherals The LTE modem supports half-duplex FDD operation and all power saving and coverage enhancement modes. A single pin antenna interface is available. The LTE stack layers L1-L3, IPv4/IPv6, TCP/UDP, TLS/DTLS are all part of the modem frmware. The application processor communicates with the LTE modem through BSD secure sockets APIs and contains the application layer protocols such as HTTP(S), CoAP, MQTT and LWM2M, and the application itself. The nRF9160 LTE modem supports both SIM and eSIM, plug-in or soldered. It provides power to the SIM and handles all communication automatically. Key data:
  • LTE-M/NB-IoT modem Frequency range 700-2200 MHz
  • Throughput (DL/UL) LTE-M: 300/375 kbps
  • Output power Up to 23 dBm
  • RX sensitivity LTE-M: -108 dBm
  • Mode HD-FDD See details in nRF9160, Product Specification v2.0 at page 360ff. Power Saving Mode (PSM) Power Saving Mode (PSM) is a state where the modem reduces its power consumption to a bare minimum to stay connected, by going into sleep mode. In PSM, the modem is not monitoring paging and as a result, becomes unreachable. To avoid the re-connection procedure when waking up again, the modem is allowed to keep its connectivity status.
    For this to happen the UE must wake up regularly and send a Tracking Area Update (TAU) to the network. This period is decided by the Periodic TAU timer. A modem in PSM will wake up for one of two reasons, either triggered by the Periodic TAU timer to send a TAU, or by the application to send uplink messages.
    Either way, the device wakes up, enters RRC Connected mode, sends and receives data packets, then stays in RRC Connected mode until the RRC Inactivity timer has run out. Then it will go to RRC Idle mode for the Active timer period to monitor paging, before going back to power saving mode.
