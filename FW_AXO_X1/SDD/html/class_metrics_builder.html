#include <metric.h>

Detailed Description

Metrics builder helper class.

Detailed Description
The metrics builder helper class provides methods to create the metrics JSON string, which will be send via LTE with topic /device/DEVICE_ID/metric/update to the MQTT broker.

◆ addHwMetrics()

void addHwMetrics ( PoolElem *  pElem)

Add function to append HW status information to metrics JSON string.

Parameters
pElemPointer to the received message (pool element)

The message payload will be extracted, casted to structure hw_metrics and stored in local varaible hw_m. The variable hw_m-> num_errors will be converted to a string and stored in private member variable num_hw_errors. The variable hw_m->hw_status will be will be checked if the value is enumeration eHwOK. In this case the the string "adxl37x_ok" will be assigned to private member variable adxl_status. If the value is not eHwOK, the string "adxl37x_fail" will be assigned to private member variable adxl_status. Writing the variable value into JSON string is part of member function finalize.

◆ isComplete()

bool isComplete ( )

isComplete function to check if the metrics JSON string is completly filled

Returns
returns true if metrics JSON string is complete, false otherwise

The function checks if the metrics JSON string is completly filled and all metrics data is present. Therefor it checks if battery voltage, temperature and internal supply voltage is present by checking taht size is greater than 0. Firthermore it checks the size of LTE temperature and humidity. Other requested metrics will not be checked. Here the implcit knowledge that LTE temperature and humidity request are the last requests, so that every other requested data still must be received.

Attention
Changing the order of requested metrics information can result in lost of metrics data.
