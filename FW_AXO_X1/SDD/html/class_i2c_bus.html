Detailed Description
The I2C bus class handles the initialization of the attached I2C slaves and the communication over the I2C bus. Major part of configuration and initialization is located in the device tree, other parts are located in the init function of this class. storage of the acceleration raw See details in Device Driver Model documentation and in API documentation

◆ init()

void init ( )

The init function.

In case initialization still has been performed, indicated by flag init_done the function returns immediately, otherwise the I2C controller device structure will be retrieved by calling macro DEVICE_DT_GET and function checks if the device is ready for usage with call of function device_is_ready. The configuration bit mask is assigned to the device by calling function i2c_configure provided by the Zephyr device driver API. The return value rc of this function, combined with the return value of function device_is_ready by logical AND will be assigned to flag device_good. In case of an error a debug message at error level will be printed. Finally init_done flag is set to true.
