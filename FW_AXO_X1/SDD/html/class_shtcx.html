Detailed Description

Shtcx temperature sensor class derived from task base class.

Detailed Description
The Shtcx class handle the reception of the temperature and humidity data and report these values to the data manager class on request to add them to the metrics JSON string. Temperature and humidity data is sampled by request. For testing purposes a cyclic timer handling is implemented, but the timer is not started. It can be activated manually in the init function to initiate a cyclic sampling of the sensor data.
The Shtcx class provides methods to read temperature and humidity data from the sensor, to initialize the sensor and to report the data to the data manager task. It uses the public zephyr API for the sensor driver provided in file sensor.h. See details in Zephyr API documentation. Message Interaction:
  • Receive
    • Timer interrupt callback
    • Shctx task (self)
    • Power manager task
  • Transmit
    • Data manager task
    • Shctx task (self, transmission in timer callback)

◆ task()

void task ( void  )
virtual

Shtcx task function.

Initially the I2C bus will be initialized by calling init function provided by I2cBus instance. Then the sensor device will be initialized by call of member function initDevice. For testing purposes a cyclic timer can be activated by commenting in the start method of the timer to initiate a cyclic sampling of the sensor data.
Afterwards the function enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On sample timer expiry sensor data will be sampled by calling function sample and the temperature and humidity data will be read out from internal driver buffer by calling the functions readTemperature and readHumidity. The pointer pElem will be set to 0, to prevent releasing the message because it is allocated static.
  • eShtcxReport: On report request sensor data will be sampled by calling function sample and the temperature and humidity data will be read out from internal driver buffer by calling the functions readTemperature and readHumidity. Afterwards the data will be send to data manager task by calling the functions reportTemperature and reportHumidity.

◆ readTemperature()

void readTemperature ( )
private

Read function to retrieve temperature from internal driver buffer.

If device has not been initialized successfully, indicated by flag device_good a defined dummy value will be written into member variable temp by calling function sensor_value_from_double provided by the zephyr API, which converts a double value to struct sensor_value and the function returns immediately. Otherwise the temperature will be read out from internal driver buffer by call of function sensor_channel_get provided by the zephyr API and the stored timestamp will be copied into structure temp of type struct shtcx_sensor_value.
