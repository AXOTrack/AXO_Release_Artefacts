Detailed Description
Metrics are send cyclically via LTE to the MQTT broker. To inform the MQTT broker about HW failure the class HW metrics collects all supported HW status information and send them to the data manager, which is responsible to send out the JSON metrics data. HW metrics provide a counter to summarize all HW errors, where
  • Counter = 0: No hardware errors detected;
  • Counter > 0: Shows number of detected hardware failures (counting per device) The following HW is supported:
  • ADXL372 acceleration sensor: Each time the device wakes up to send metric data the sanity of the adxl372 sensor should be checked by reading the status registers. The check should be made before metric data are sent.
