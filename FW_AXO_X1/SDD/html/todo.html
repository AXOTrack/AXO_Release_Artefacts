Class LteMonitor
RMS task receives the acceleration raw data from ADXL data store task, calculate RMS and with the help of class HitDetector hit values. RMS, medium and critical hit values will be send to RMS store task together with a timestamp. The class provides an idle detection timer, and for medium and critical hit detection an instance of class HitDetector. It provides the function update_rms to calculate RMS and hit values and to send them to the RMS store task.

enable raw data transmission trigger from configuration HTTP connector task send eAdxlIdleReq to the Adxl37x task. If the returned indication signals Idle = false, the delay timer will be started. After delay timer expiry the idlse status of Adxl37x task will be requested again until the response indicated Idle = true. Thean HTTP connector task starts data transmission by sending eHttpRawDataSend message to its own mailbox and inform Adxl37x task about active data transfer by sending eRawDataTransferActive. Afterwards HTTP connector task opens a POSSIX conform socket stream and establish the HTTPS connection. See Message Sequence Raw Data Transmission for further processing.

enable raw data transmission trigger from configuration HTTP connector task send eAdxlIdleReq to the Adxl37x task. If the returned indication signals Idle = true, HTTP connector task starts data transmission by sending eHttpRawDataSend message to its own mailbox and inform Adxl37x task about active data transfer by sending eRawDataTransferActive. Afterwards HTTP connector task opens a POSSIX conform socket stream and establish the HTTPS connection. If connection is established, the HTTP POST Header will be send to remote JOVIA webserver and the raw data status will be requested from ADXL store task. The received response eRawDataStatus indicating that total number of raw data entries is 0 and the Raw Data Header will be constructed with IMEI, timestamp and number of raw data entries = 0 and send to JOVIA webserver in the first chunk frame. Afterwards the end of chunk frame will be send and the Adxl37x task will be informed that data transmission has been completed with message eRawDataTransferComplete. Finally the socket stream will be closed.

Adapt to MRAM !!!!!! At each step the input data needs to read from FRAM in data packets with less than 1024 bytes. These data packets must be requested by the Displacement Task (DispTask) from ADXL Store Task. The Displacement Task calculates the output data for every data packet and send the calculated output values to the ADXL Store Task, which store it in FRAM. The FRAM is separated into three data blocks, First block is reserved for raw data (x, y, z), the second block is used for forward filtered data and S result values and the last block is used for backward filtered data and final D result values. See section fram for details of the FRAM memory layout.

The calculated displacement data will be transmitted via HTTPS protocol. Transmission uses HTTP POST. See http_connector.

File mram.h
Create memory map !!!!!!! In one measurement cycle (15 measurements in an one second cycle with sampling rate of 4400Hz) the capacity of the FRAM is nearly exhausted (). Therefore a measurement cycle has been limited to 15 seconds.
To be able to calculate displacement data the FRAM raw data storage must be reduced to 5 seconds measurement, so that there is enough space to store intermediate and final displacement data. The memory layout is therefore separated in three blocks (see section Displacement Calculation for further details).
