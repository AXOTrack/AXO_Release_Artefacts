RTOS wrapper for message pool support implemented according to C++ singleton design pattern.

Detailed Description
The MsgPool class implements a C++ wrapper for message pool support using the kernel memory slab API provided by the Zephyr operating system. See details in Zephyr Memory Slab documentation or in API documentation. The MsgPool class provides methods to allocate messages from the pool and to release (free) them back into the pool. Furthermore it provides a method to count the number of objects in the queue and to print statistics.

◆ getNewElem() [1/2]

PoolElem * getNewElem ( int  minSize)

Function to allocate a message of type PoolElem from message pool.

Parameters
minSizeThe minimal size of the message buffer
Returns
A pointer to the allocated message of type PoolElem

The function lock mutex lock to avoid concurrent access and allocate a message by calling function alloc of the message pool elemPool of class MemoryPool. According to the minimal size of the message buffer minSize the buffer will be allocated from one of the fixed sized buffer pools and the member variables buf and bufLen of the PoolElem class are set accordingly. Finally the member variables of the PoolElem class are assigned. Variable dataLen to 0, variable pool with this pointer of MsgPool instance and variable inPool to false before the pointer to the message of type PoolElem will be returned.

void freeElem ( PoolElem *  elem)

Function to free (release) a message of type PoolElem to the message pool.

Parameters
elemPointer to the allocated message of type PoolElem

The function lock mutex lock to avoid concurrent access. According to the size of the buffer the message buffer will be released to the fixed size buffer it belongs to by calling the fixed size pools free method. Afterwards the message will be released by cvalling the message pool method free and variable inPool will be set to true.
