Reset Control class derived from task base class.

Detailed Description
The Reset Control class implements the cyclic power cycle reset (cold start). It controls Pin P0.01 of nRF9160 which can trigger the reset. The cycle period is controlled by a timer.
The Reset Control class provide a timer callback function and a timer expired function, used to transform external timer expiry event from interrupt context to internal timer expired message of type PoolElem. Furthermore an initialization function and a trigger function, which perform the cold start reset. Message Interaction:
  • Receive
    • Timer interrupt callback
    • Reset control task (self)
  • Transmit
    • Reset control task (self)

◆ task()

void task ( void  )
virtual

Reset control task function.

Initially the cyclic timer for power cycle reset (cold start) will be started. Afterwards the function enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On metrics TX interval timer expiry the pointer pElem will be set to 0, to prevent releasing the message because it is allocated static. Finally cold start will be initiated by calling function trigger. Since FW 1.74 cold start has been replaced by SW reset.
  • eColdStartReq: A cold start will be initiated by calling function trigger.

Implements Task.

◆ performSwReset()

void performSwReset ( )
static

Convenience function to perform a SW reset.

The function disconnect from MQTT broker by calling function mqtt_helper_disconnect provided by NRF MQTT helper library, followed by two calls of function nrf_modem_at_printf provided by nrfxlib API with the AT commands 'AT+CFUN=0' and 'ATXFACTORYRESET=0' to disable the reset loop detection temporarily. Finally a SW reset will be performed by call of function sys_reboot.
