Firmware download state class derived from base class FwState.

Detailed Description
The firmware download state class overrides the handler function of DlComplete, DlFailed and timer expired events. In this state the download of the firmware binary file is ongoing.

◆ handle_DlFailed()

FwFsm::StateId handle_DlFailed ( PoolElem **  pElem)
virtual

Firmware download state handler function for DlFailed events.

Parameters
pElemPointer to pointer to message of type PoolElem
Returns
Returns the state ID of the idle state or eNoChange

The function checks if the maximum number of retries has not been reached. In this case the function start the retry timer by calling function startRetryTimer of the FwUpdCtrl class instance to schedule a firmware download retry and returns the state ID eNoChange. If the maximum number of retries has been reached the retry counter will be set to 0 and the DlFailed message will be forwarded to MQTT connector task. Afterwards the function performes a delay of 2 seconds to let other tasks take over process, before it performes a reset. Returning to the state ID eIdle will never be reached.

◆ handle_timer_exp()

FwFsm::StateId handle_timer_exp ( )
virtual

Firmware download state handler function for timer expired events.

Returns
The state ID eNoChange

The function calculated the time difference from the download start time to the current time. If the time difference is more than maxDownloadDelay or if the maximum number of retries has been reached the download will be expected as failed and message eFwDlFailed will be send to MQTT connector task with error CODE3 and after a delay of 2 seconds a reset will be performed.
Otherwise a new download attempt will be started by calling the method requestDownload of the FwUpdCtrl class instance. Pointer variable pd is assigned to FwUpdCtrl class instance.
