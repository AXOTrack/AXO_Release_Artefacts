Detailed Description
The application is implemented as a preemptive multithreading system, separated into tasks with underlaying RTOS thread API. Every task is a single process which is normally in a blocking state and wait for activation. Every task owns a mailbox, which is called in an endless loop and blocks the process. The activation of the task is realized by messages (events), which will be received in the task's mailbox. Messages are send to the mailbox from other tasks or from interrupt context. On receiving the message the task wakes up and the message will be processed, forwarded to another task or delegated to a state machine that handles the event. After processing the message the task enters again blocking state and wait for further messages.
