RmsStore class derived from task base class.

Detailed Description
The RMS storage class is responsible to store the received RMS and hit values from RMS task and to forward them to the data manager task. On receiving a RMS report request from power manager task, the request shall be also forwarded to the data manager task. The RMS storage class provides a store and reset method and methods to print out the data for testing purposes. Message Interaction:
  • Receive
    • RMS task
    • Power manager task
    • RMS storage (self, callback function for shell command 'r')
  • Transmit
    • Data manager task
    • RMS storage (self, callback function for shell command 'r')

◆ task()

void task ( void  )
overridevirtual

RMS storage task function.

The function directly enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eAdxl37xRmsData: If RMS storage task is in idle state, indicated by flag idle, flag idle is set to false and stored data will be deleted by calling member function reset. Afterwards the received data will be extracted from messages payload into structure of type struct adxl_rms_data and stored in vector by calling member function store. If the vector was not full, indicated by flag full, the data will be forwarded as it is to the data manager task and the pointer to message pElem is set to 0 to prevent releasing the message back to the pool.
  • eAdxl37xIdle: The flag idle will be set to true and the last entry of the currently stored data will be printed out by call of member function dump.
  • eRmsReport: The message will be forwarded as it is to the data manager task and the pointer to message pElem is set to 0 to prevent releasing the message back to the pool.
  • eAdxlStorageDumpAll: All currently stored data will be printed out by call of member function dumpAll.

    For more details see also message sequence charts in section Measurement
