RTOS wrapper for multithreading support.

Detailed Description
The task class implements a C++ wrapper of the thread API provided by the Zephyr operating system. See details in Zephyr thread documentation or in API documentation. The task class is the base class of all task in the AXO track system. All tasks shall be derived from this base class. Every task has it's own stack, which needs to be created outside of the task and an assigned priority. The task class provides a method to create and to start the task and to assign an unique ID. It provides a virtual task function, which needs to be implemented in the derived classes and shall implements the endless loop. Furthermore it provides the entry point for the operating system, which simply calls the task function.

Entry point for the operating system.

Parameters
p1First argument

This function realize the entry point for the Zephyr operating system of type k_thread_entry_t. A thread's entry point function is invoked when the thread starts executing. Up to 3 argument values can be passed to the function. The thread terminates execution permanently if the entry point function returns. The thread is responsible for releasing any shared resources it may own (such as mutexes and dynamically allocated memory), prior to returning. See detail in API documentation

Function to create a task and to assign the name of the task.

Parameters
stackPointer to stack object of type k_thread_stack_t
stack_sizeSize of the stack in bytes
prioThe priority of the task

This function create a task, assign the stack and the priority and return the unique ID of the task provided by the OS of type k_tid_t and the thread administration structure thread_handle of type struct k_thread by calling the function k_thread_create provided by the operating system. Afterwars the task's name will be assigned by calling function k_thread_name_set also provided by the operating system.
Priority: The smaller the value, the higher the priority
