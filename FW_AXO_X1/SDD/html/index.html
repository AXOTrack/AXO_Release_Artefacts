1.2. Scope

This document is intended for the people programming and testing the software of the AXO X1.

2. Project Environment and Tools

2.1. Hardware

  • *Microchip**
    AXO X1 hardware uses nRF9160 microchip from Nordic Semicondutor with integrated LTE modem
    Acceleration Sensor
    Acceleration Sensor used is ADXL372 from Analog Devices

Humidity and Temperature Sensor
Humidity and Temperature Sensor used is SHTC3 from Sensirion Sensor Company

FRAM
FRAM (Ferroelectric Random Access Memory) chip used is Fujitsu Semicondutor MB85RS4MTY with 512 KByte memory and SPI interface

Boost Converter
As boost converter XC9145 step-up DC/DC converter of Torex Semiconductor is used

Schematic Prometheus_for_AxoTrack_Mk5.4rn.pdf

2.2. Operating System

As operating system Zephyr RTOS is used in AXO X1 project, used version is 3.2.99.

2.1. Software development tools

Visual Studio Code is used as development environment. The environment can be seup in a docker container or plain on an Linux Ubuntu 10.04 computer. Environment setup process is explained in detail AXO GitHub repository main page

Note
Setting up the docker container under windows will fail, because link generation is part of the setup process and is designed only for Linus links.

2.3. Test Tools

AXO X1 firmware uses the Zephyr test framework Ztest. For code coverage gcc tool gcov is used.
