Detailed Description

HTTP connector class derived from task base class.

Detailed Description
HTTP connector task handle the HTTPS connection to send raw data to the remote JOVIA webserver. It uses the BSD socket API provided by Zephyr OS.
See details of BSD socket API The HttpConnector class provides functions to request connection, to send raw data via HTTP POST and to receive the server responses. It provides suppport of Transport layer security (TLS) and furthermore provides functions to request and receive the raw data from ADXL Store task, to request the status of the ADXL Store data and to inform ADXL task about running or stopped raw data transmission. Message Interaction:
  • Receive
    • Timer interrupt callback
    • MQTT connector task (self)
    • LTE connector task
    • LTE monitor task
    • ADXL store task
  • Transmit
    • ADXL store task
    • ADXL task
    • HTTP connector task (self, callback function for shell command 'z')

◆ task()

void task ( void  )
virtual

HTTP connector task function.

The function directly enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On delay timer expiry the ADXL store idle status will be checked again by sending message of type eAdxlIdleReq to AdxlStore task by call of function sendMsg provided by Mbox class. Afterwards the pointer pElem will be set to 0, to prevent releasing the message because it is allocated static.
  • eOneDayTimerExpired: The counter for raw data sendings per day dailyRawSendCnt will be reset to 0 and the pointer pElem will be set to 0, to prevent releasing the message because it is allocated static.
  • eLteSleepEnter: Will be ignored, only debug message at info level will be printed.
  • eLteSleepExit: Will be ignored, only debug message at info level will be printed.
  • eLteConnected: If LTE is not connected, the flag lte_connected will be set to true. Furthermore Adxl37x task will be requested to enable measurement by call of function sendMsg with message type eAdxlResumeMeas.
  • eLteDisconnected: The flag lte_connected will be set to false and the member function http_disconnect will be called.
  • eLteIMEI: The IMEI will be extracted from message payload and stored in member variable imei.
  • eTimestamp: Timestamp will be extracted from message payload and stored in member variable timestamp
  • eRawDataStatus: Member function HandleRawDataStatus will be called with message pointer pElem as parameter.
  • eRawData: Member function HandleRcvdRawData will be called with message pointer pElem as parameter.
  • eAdxlIdleReq: The received message of type eAdxlIdleReq will be forwarded to AdxlStore task by calling function send provided by Mbox class. Afterwards the pointer pElem will be set to 0, to prevent releasing the message. Releasing the message to the pool is task of the AdxlStore task.
  • eAdxl37xRmsData: Structure Adxl_xyz_max_data will be extracted from message payload and the maximum value in Z direction will be calculated and stored in variable zMaxRms. Finally the received message will be forwarded to data manager task and the pointer pElem will be set to 0, to prevent releasing the message.
  • eAdxl37xIdle: Member variable zMaxRms is set to 0.
  • eAdxlIdleInd: Member function HandleAdxlIdleInd will be called with message pointer pElem as parameter.
  • eHttpRawDataSend: If LTE is connected and raw data tranfer is enabled, provided by the dynamic configuration the raw data transmission will be started by call of member function StartHttpRawDataTransmission, otherwise a debug message at error level will be printed.
  • eSendDispDataReq: If LTE is connected and raw data tranfer is enabled, provided by the dynamic configuration the displacement data transmission will be started by call of member function StartHttpDispDataTransmission, otherwise a debug message at error level will be printed and Adxl37x task will be requested to enable measurement by call of function sendMsg with message type eAdxlResumeMeas.
  • eDispData: Handle the received displacement data (calculate SHA1 hash, send data to remote server, request next data) by calling member function HandleRcvdDispData.

◆ http_connect()

void http_connect ( )

Function to create a socket stream and send a HTTP connect request.

The function call the getaddrinfo function provided by Zephyr BSD socket API, which itself is a POSIX wrapper for zsock_getaddrinfo (see details) to retrieve a set of socket addresses and associated information to be used to create a socket. The socket will be created by call of the function socket provided by Zephyr BSD socket API, which itself is a POSIX wrapper for zsock_socket (see details). The returned file descriptor client_fd is then used to setup Transport Layer Security (TLS) if requested protocol is of type IPPROTO_TLS_1_2. Finally member function blocking_connect is called to connect the device to the HTTP host. On success the flag http_connected will be set to true, otherwise function http_connect close the socket and returns immediately.

void SendHttpPostHeader ( eDataType  dataType)
private

Function to send HTTP POST header to the host.

The function copies the header information into the send buffer and enters a loop. In the loop member function blocking_send is called until all data in send buffer has been transmitted. The header contains the following information:
POST / HTTP/1.1(CR)(LF)
Host: HOSTNAME+HOSTPATH(CR)(LF)
Content-Length: PAYLOAD-LEN(CR)(LF)" \n Content-Type: application/octet-stream(CR)(LF)"
(CR)(LF) Where PAYLOAD-LEN is the length of the data header + length of data bytes + length of SHA1 hash (20 bytes)

◆ HandleRcvdData()

bool HandleRcvdData ( PoolElem *  pElem,
uint16_t  dataSize 
)
private

Function to handle received data from AdxlStore task.

Parameters
pElemPointer to the received message
Returns
Returns boolean variable bLastFrame, indicating last frame has been send or not

The boolean variable bLastFrame will be initialized with false and the number of still received raw data entries in member variable rcvdEntries will be increased by the number of requested raw data entries. If the total number of entries in adxlStore instance is greater than 0, the SHA1 checksum will be updated with the received raw data by calling function update provided by class Hash, than the received raw data will be send to host in a data frame by calling member function SendSingleDataFrame with the raw data in the message payload and the size of the payload calculated by number of requested entries multiplied with number of bytes of a single raw data entry.
If all raw data has been sent, when number of received entries is equal to total number of entries the SHA256 hash checksum will be finalized by calling function finalize provided by class Hash and the signature of the hash checksum will be calculated by call of function signHash provided by class ECDSASigner. The signature will then be send to remote host by calling member function SendSignatureFrame. With call of function blocking_recv the response from remote server will be received, the socket will be closed by calling function close provided by Zephyr BSD socket API and boolean variable bLastFrame will be set to true. Finally bLastFrame will be returned.

Function to handle idle indication from AdxlStore task.

Parameters
pElemPointer to the received message

The method checks, if raw data transmission is enabled by configuration and in this case it extract the idle flag from message and start a delay timer if idle flag is false indicating that raw data recording is still in progress. Otherwise it checks if current maximum RMS z value is greater than RMS z threshold and maximum number of raw data sendings per day is not exceeded. Iin this case it starts the raw data transmission by sending message of type eHttpRawDataSend to own (HttpConnector's) mailbox by call of function sendMsg, provided by the Mbox class, otherwise it send message eAdxlResumeMeas to eAdxl37x Task to resume the measurement. If raw data transmission is not enabled, the function checks if displacement calculation is enabled and if current maximum RMS z value is greater than RMS z threshold. In this case it send the message eCalcDisplacementReq to the disp task by calling function sendMsg to request the start of displacement calculation, otherwise it send message eAdxlResumeMeas to eAdxl37x Task to resume the measurement.
