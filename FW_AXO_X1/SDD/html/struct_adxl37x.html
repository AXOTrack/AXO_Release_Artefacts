#include <adxl37x.h>

Detailed Description

Adxl37x acceleration sensor class derived from task base class.

Detailed Description
The Adxl37x class handle the reception of the acceleration data, delegate storage and further processing to Adxl data store task. It provides methods to receive the data from driver FIFO, to initialize the Adxl372 driver, to set the operating mode of the driver, to switch synchronization mode between internal and external, enable and clear driver FIFO, to convert and dump the received data and to inform Adxl data store task about new data.
Note
To avoid instabilities of the system when OTA Message Interaction:
  • Receive
    • Adxl data store task
    • RMS task
    • Firmware update control task
    • HTTP connector task
    • Adxl37x task (self, from callback function fifo_handler)
  • Transmit
    • Adxl data store task
    • HTTP connector task
    • Adxl37x task (self, transmission in callback function fifo_handler)

◆ task()

void task ( void  )
virtual

Adxl37x task function.

The task function directly enter the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eAdxl37xFifo: Indication, that sensor raw data in FIFO available. The data will be send to Adxl data store task by calling function report. Afterwards pElem is set to 0 to avoid free() of statically allocated message.
  • eAdxl37xIdle: Operation mode of ADXL372 sensor will be set to Standby by calling function suspend. FIFO is cleared by calling setFifoBypass and activated for further receiption by calling setFifoOn. Finally Adxl data store task will be informed about idle state.
  • eAdxlStorageDone: Previous used buffer will be marked as clean
  • eAdxlStorageFull: Operation mode of ADXL372 sensor will be set to Standby by calling function suspend to supress measurements.
  • eAdxlResumeMeas: Operation mode of ADXL372 sensor will be set to WakeUp by calling function resume to enable further measurements.
  • eFwUpdateActive: Indication that an OTA firmaware update is started. Operation mode of ADXL372 sensor will be set to Standby by calling function suspend to supress measurements during SW update.
  • eRawDataTransferActive: Indication that transfer of raw data is running. Only informational, triggers a debug message.
  • eRawDataTransferComplete: Indication that transfer of raw data is finished. Only informational, triggers a debug message.
  • eDispDataTransferActive: Indication that transfer of displacement data is running. Only informational, triggers a debug message.
  • eDispDataTransferComplete: Indication that transfer of displacement data is finished. Only informational, triggers a debug message.
  • eAdxlStorageDumpActive: Indication that console data dump is running. Operation mode of ADXL372 sensor will be set to Standby by calling function suspend to supress measurements during data dump.
  • eAdxlStorageDumpComplete: Operation mode of ADXL372 sensor will be set to WakeUp by calling function resume to enable further measurements.
  • eHwReport: Report the HW status to the mailbox of HW metrics task

◆ initDevice()

void initDevice ( )
private

Function to initialize the sensor device driver.

Function initializes the SPI bus by calling function spibus::inst()->init() and retrieves a pointer to the sensor device object by calling macro DEVICE_DT_GET_ANY and casting to pointer of type struct device. With call of function device_is_ready it checks if the sensor device is ready to use. Result is stored in private class variable device_good. If device is not ready an error message will be printed. Furthermore the operating mode of the device is set to STANDBY mode, trigger type and channel are assigned and set. Afterward external syncronization signal will be initialized with 4400 Hz and 80 usec pulse width by calling Pwm::inst()->init(), Pwm::inst()->setPeriod() and Pwm::inst()->setPulseWidth().
Afterwards FIFO will be cleared and activated and operation mode is set to FULL_BW_MEASUREMENT and STANDBY. This sequence triggers the initial calibration of the sensor to the current prevalent acceleration ratio in idle mode of the sensor.
Finally FIFO will be cleared and activated and operation mode is set to WAKE_UP. See ADXL372 datasheet, page 19ff for interrupt configuration, page 25 for interrupt configuration and page 29 for SPI support.

Note
SPI clock frequency has to be adjusted in file nrf9160dk_switch_common.dts with entry spi-max-frequency = <8000000>; The value must be unsigned integer in Hz.

In Zephyr devicetree ADXL SPI is SPI bus 2.
