Measurement builder helper class.

Detailed Description
The measurement builder helper class provides methods to create the measurement JSON string, which will be send via LTE with topic /device/DEVICE_ID/measurement/update to the MQTT broker.

Add function to append RMS data to measurement JSON string and store hit counter.

Parameters
pElemPointer to the received message (pool element)

The message payload will be extracted, casted to structure adxl_rms_data and stored in private member variable adxl_rms_data. The function calculate the highest RMS value of the complete measurement and store it in member variable max_rms_z. The hit counter for medium and critical hits will be appended to the vectors med_hit_cnt and crit_hit_cnt and the x, y and z RMS values will be appended to the measurement JSON string variable payload. In case it is the first measurement data the message header will be written in front of the first RMS entries.

◆ finalize()

void finalize ( void  )

Function to finalize the measurement JSON string.

The function append the medium hit counter in vector med_hit_cnt to private string variable payload, which represets the measurement JSON string. Afterwards it append the critical hit counter in vector crit_hit_cnt to private string variable payload, before appending the x, y and z maximum values and the displacement minimum, maximum and peak to peak statistic values to string variable payload. The metrics JSON string will be terminated with closing square and curly bracket. Finally member function print will be called to output the measurement JSON string with debug log level INFO.

void reset ( )

Function clear all private string variables related to metrics JSON string.

The function initialize member variable max_rms_z with 0, clear the vectors med_hit_cnt and crit_hit_cnt for medium and critical hit counter. Variable payload will be reinitializes with opening curly bracket and entry time_interval with fixed value of 1000 ms (1 sec) and variable timestamp will be initialized with an empty string. ◆ isComplete()

bool isComplete ( bool  disp_mode_off,
double  rmsZThres 
)

isComplete function to check if the measurement JSON string is completely filled

Parameters
disp_mode_offFlag indication if displacement calculation is disabled
rmsZThresActivation threshold for displacement calculation and raw / displacement data transmission
Returns
returns true if measurement JSON string is complete, false otherwise

The function checks if displacement calculation is disabled, flag disp_mode_off is true or maximum RMS z value is lower or equal to RMS z threshold. In this case it checks if the size of measurement JSON string is greater than 30 bytes. If size is greater it is ensured, that at least one measurement entry is present and the function returns true indicating, that JSON string is completely filled. If displacement calculation is enabled the function compairs the content of the string sDispP2P with the string sVal, which contains the value of the VAL_NOT_INITIALIZED_FLOAT constant. If the strings are equal, the displacement calculation is still ongoing and the function returns false. If the strings are not equal, the displacement values are still calculated and stored in measurement JSON string and the function returns true indicating, that JSON string is completely filled.
