Detailed Description

Adxl data store class derived from task base class.

Detailed Description
The adxl data store class reserves FRAM space capacity for samples within 15 seconds at expected sample rate of 4400 (samples per second). It provides methods to store the raw data, to reset/clear previously stored data, to dump the data and to indicate that storage is full or free for re-usage.
Message Interaction:
  • Receive
    • Adxl37x task
    • RMS task
    • Adxl data store (self, callback function for shell command 'd')
  • Transmit
    • Adxl37x task
    • RMS task
    • RMS store task
    • Adxl data store (self, callback function for shell command 'd')

◆ task()

void task ( void  )
virtual

Adxl data store task function.

Initially the SPI bus instance andd the FRAM instance will be initialized by calling the functions spibus::inst()->init() and MEMORY_INST->init(). Afterwards FRAM instance will be forced in suspended/power saving mode by calling MEMORY_INST->sleep() and the power manger state will be set to off, before entering the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eAdxl37xSensorData: Sensor raw data. If in idle state flag idle is set to false and function reset is called to be able to overwrite old data from last measurement cycle. Payload of message will be extracted and casted to structure adxl37x_sensor_value before message is forwarded to RMS task, message pointer pElem is set to 0 to prevent free(). Afterwards the maximum acceleration values in x, y and z direction will be calculated by calling member function calcMaxXYZ. If power manager state is eOff, FRAM will be activated by calling function MEMORY_INST->wakeup and power manager state is set to eOn. Data is storen in FRAM by calling funtion store and Adxl37x task will be informed with message eAdxlStorageDone by calling funtion notifySender.
  • eAdxl37xIdle: Flag idle will be set to true, FRAM will be forced to low power state by calling function MEMORY_INST->sleep() and power manager state is set to eOff. Afterwards the received message will be forwarded to RMS store task and message pointer pElem is set to 0 to prevent free(). In case have_http_connector is true, the calculated maximum acceleration values in x, y and z direction will be send to data manager task and finally the HTTP connector will be informed, that raw data is available by sending message eAdxlIdleInd.
  • eAdxlStorageDumpAll: Implemented for testing purposes. In case user input the command 'd' on terminal raw data will be printed. FRAM will be activated by calling function MEMORY_INST->wakeup and raw data will be printed by calling function dumpAll.
  • eAdxlStorageDumpBytes: Implemented for testing purposes. In case user input the command 'db' on terminal raw data bytes will be printed. FRAM will be activated by calling function MEMORY_INST->wakeup and raw data will be printed by calling function dumpAllBytes.
  • eAdxlStorageDumpFilt: Implemented for testing purposes. In case user input the command 'df' on terminal filtered values will be printed. FRAM will be activated by calling function MEMORY_INST->wakeup and raw data will be printed by calling function dumpAllFilt.
    Note
    only for development phase, inactive for relaese versions.
  • eAdxlStorageDumpDisp: Implemented for testing purposes. In case user input the command 'dd' on terminal displacement data will be printed. FRAM will be activated by calling function MEMORY_INST->wakeup and raw data will be printed by calling function dumpAllDisp.
  • eAdxlIdleReq: The value of the idle flag will be send to HTTP Connector task with message of type eAdxlIdleInd and sender ID eAdxlStore.
  • eRawDataStatus: Call of handler function notifyRawDataStatus.
  • eRawData: Payload of the received message will be casted to structure of type AdxlRawDataProgress and function sendRawData will be called.
  • eDispFirstEntries: same as eRawData
  • eDispLastEntries: same as eRawData
  • eFwFiltData: Send forward filtered values to requesting task by calling member function sendFwFiltData
  • eBwFiltData: Send backward filtered values to requesting task by calling member function sendBwFiltData
  • eSResultData: Send S values (displacement before detrending) to requesting task by calling member function sendSResultData
  • eStoreFwFiltResult: Write forward filtered data into FRAM by calling member function storeForwardFilt
  • eStoreBwFiltResult: Write backward filtered data into FRAM by calling member function storeBackwardFilt
  • eStoreSResult: Write S values (displacement before detrending) into FRAM by calling member function storeSResult
  • eStoreDResult: Write D values (displacement after detrending) into FRAM by calling member function storeDResult (this is final displacement result)
  • eDispData: Send displacement data to the requesting sender task by calling member function sendDispData

◆ store()

void store ( adxl372_xyz_accel_data *  src,
uint32_t  cnt 
)

Adxl data storage function.

Parameters
srcpointer to sample data of type adxl372_xyz_accel_data
cntnumber of samples

The function checks if FRAM is full with private member variable full. In this case the function returns immediatly. Otherwise the function calculates the needed number of bytes for the new samples. In case write position wr plus needed number of bytes for the new samples exceeds size of FRAM capacity only the number of samples fitting into FRAM will be written, write position wr and remaining bytes to be written in variable len will be actualized and the indicator that FRAM is full will be set to true. eAdxl37xTask will be informed with message eAdxlStorageFull by calling function notifyStorageFull. Furthermore write position wr will be set to 0.
If number of bytes to be written in variable len is greater 0 and FRAM is not full or allow_wrap_around is true, samples will be written to FRAM and write position wr will be actualized by adding number of written bytes len. For debug purposes a test is implemented, which check if 1 sec data is available.

◆ dump()

void dump ( )

Function prints out the data of the last second.

The function enters a loop with current read position rd as start point. Read position will be incremented by size of one data entry sizeof(adxl372_xyz_accel_data) and loop ends if read and write position are equal. In the loop raw data will be read from FRAM at current read position by calling function MEMORY_INST->readBytes, data will be converted to x, y, z values and these values will be printed by calling function printk. Finally a sleep for 1 msec will be performed by calling function k_sleep.

Note
The function do not handle wrap around

◆ dumpAll()

void dumpAll ( )

Function prints out all the data stored in FRAM.

The function set current position pos and end position end. If FRAM is not full, all the written data shall be mentioned (pos is 0 and end is the current write position). If FRAM is full and wrap around is not allowed, complete FRAM shall be mentioned (pos is 0 and end is the size of FRAM capa). If FRAM is full and wrap around is allowed, also complete FRAM shall be mentioned (pos is the current write position and end is the current write position plus the size of FRAM, wrap around is handled in loop) Afterwards the function calculates the number of data entries and enters the loop over the number of data entries. After every loop cycle the current read position pos is incremented by size of one data entry sizeof(adxl372_xyz_accel_data) modulo size of the FRAM. The modulo operation realize the wrap around. In the loop raw data will be read from FRAM at current read position by calling function MEMORY_INST->readBytes, data will be converted to x, y, z values and these values will be printed by calling LOG_PRINTK. Finally a sleep for 4 msec will be performed by calling function k_sleep. *

◆ dumpAllFilt()

void dumpAllFilt ( )

Function prints out all the data stored in FRAM.

The function set current positions pos to the start address of reserved memory for forward filtered and S (displacement data before detrending) values and pos2 to the start address of reserved memory for backward filtered and D (displacement data after detrending / final displacement result) values. Afterwards the function enters the loop over the number of samples. After every loop cycle the current read positions pos and pos3 are incremented by size of one data entry sizeof(double). In the loop forward and backward filtered data values will be read from FRAM at current read position by calling function MEMORY_INST->readBytes, data will be converted to double and printed by calling LOG_PRINTK. Finally a sleep for 10 msec will be performed by calling function k_sleep.

Note
Function is only for testing purposes during development!

◆ configure()

void configure ( )
virtual

Adxl DataStore virtual task configure function.

The configure function overloads the configure function of the task base class. It retrieves the configuration related to ADXLDataStore by call of the function getCfgDatalog provided by the Config class, checks if the displacement calculation is configured to be disabled and set the number of FRAM storage time to 15 seconds in this case, otherwise FRAM storage time is set to 5 seconds. Finally the total number of required bytes are calculated and stored in member variable capa.

void storeSampleData ( )
private

Helper function to store known defined acceleration data in FRAM.

The function includes the known defined acceleration data in header file input_accel_bin.h, wake up SPI bus by calling function wakeup of class spi(f/m)ram, assign the write position wr to address 0 and enter a loop over the number of samples to be written. In the loop every single entry is written to write position by call of function writeBytes of class spi(f/m)ram and write position will be incremented by sizeof(adxl372_xyz_accel_data).

Note
Only the values in z direction will be written
This funtion is implemented only for development purposes. In release version of the firmware the function is never called.
