Detailed Description

Firmware update task class derived from task base class.

Detailed Description
The firmware update task implements the interface to the FOTA download library. It starts the download process and receives the FOTA download event. It provides a callback function to receive and process the FOTA events, a function to start the download process and functions to call on download success and download failure. Furthermore two setter functions to set the name of the host, which is the server to download from and name of the binary file to be downloaded.
  • Receive
    • FOTA download events via callback function
    • Firmware update control task
  • Transmit

◆ task()

void task ( void  )
virtual

Firmware update task function.

Initially the task function calls function boot_write_img_confirmed provided by the MCUboot API. This marks the currently running image as confirmed, which will install it permanently, preventing MCUboot from reverting it for an older image at the next reset.
Afterwards the function fota_download_init, provided by the FOTA download library of the nRF Connect SDK, will be called to initialize the firmware over-the-air download library.
The function than enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eFwInfo: The function allocates an object of type FwInfo and call its method parse to parse the JSON string and to extract the download parameters. To check the validity of the parameters method validate will be called. It extracts the size of the firmware binary and the sha1 checksum and write to download library with call of function fota_download_set_expected provided by the FOTA download library ofthe nRF Connect SDK. With a call of method splitUrl of the FwInfo instance the received URL will be split into host (the http server to download from) and filename (File to request in download) and both will be stored in the member variables hostname and filename by calling the appropriate setter functions. Finally the download will be started by calling the member function download_start.
  • eFwDownload: Download will be retried because of a previous failure by calling the member function download_start.

Callback function to receive the FOTA download events.

Parameters
evtPointer to the event of type struct fota_download_evt

The function checks the event ID. In case the event ID is FOTA_DOWNLOAD_EVT_ERROR or FOTA_DOWNLOAD_EVT_CANCELLED member function download_failed will be called with an appropriate error code. In case the event ID is FOTA_DOWNLOAD_EVT_FINISHED member function download_success will be called. On unexpected event IDs a debug message at error level will be printed.

◆ download_start()

void download_start ( void  )
static

Function to start the OTA download process.

The function start the OTA download process by calling function fota_download_start provided by the FOTA download library of the nRF Connect SDK. Function fota_download_start start downloading the given file from the given host. When the download is complete, the secondary slot of MCUboot is tagged as having valid firmware inside it. The completion is reported through an event (see fota_dl_handler). In case of an error, indicated by a return value unequal to 0, the member function download_failed will be called.
