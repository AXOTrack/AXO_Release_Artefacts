DataManager class derived from task base class.

Detailed Description
The data manager class is responsible to collect metrics and measurement data, to construct the metrics and measurement JSON strings with support of the metrics builder class and the measurement builder class and to publish metrics and measurement JSON strings by sending them to the MQTT connector task. It provides two publish methods, one for metrics and the other one for measurement data. Message Interaction:
  • Receive
    • LTE monitor task
    • LTE connector task
    • Power manager task
    • AD converter task
    • Shtcx task
    • HW metrics task
  • Transmit
    • MQTT connector task

◆ task()

void task ( void  )
overridevirtual

Data manager task function.

The function directly enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eLteReport: Received report request, change sender to eDataManager and forward request to LTE monitor task. The pointer pElem will be set to 0, to prevent releasing the message.
  • eLteLinkQuality: Add payload to metrics JSON string by calling function add provided by metricsBuilder class. In case metrics JSON string is complete, finalize and publish it by calling function finalize of metricsBuilder class and member function publishMetrics. Finally metrics JSON string will be cleared by calling function reset also provided by metricsBuilder class.
  • eLteBattVoltage: same as eLteLinkQuality
  • eLteTemperature: same as eLteLinkQuality
  • eLtePsmAct: same as eLteLinkQuality
  • eResetCause: same as eLteLinkQuality
  • eBootCnt: same as eLteLinkQuality
  • eAdcData: same as eLteLinkQuality
  • eShtcxTemperature: same as eLteLinkQuality
  • eShtcxHumidity: same as eLteLinkQuality
  • eLteIMSI: same as eLteLinkQuality
  • eLteFwVersion: same as eLteLinkQuality
  • eHwMetrics: same as eLteLinkQuality
  • eAdxl37xRmsData: Add RMS values to payload variable of class measurement and store hit values in member varaibles of this class
  • eDispStatistics: same as eRmsReport
  • eMaxXYZ: Add x, y and z maximum values to member variables of measurement class
  • eRmsReport: In case measurement JSON string is complete, finalize and publish it by calling function finalize of measurementBuilder class and member function publishMeasurements. Finally measurement JSON string will be cleared by calling function reset also provided by measurementBuilder class.
  • eDispStatistics: In case measurement JSON string is complete, finalize and publish it by calling function finalize of measurementBuilder class and member function publishMeasurements. Finally measurement JSON string will be cleared by calling function reset also provided by measurementBuilder class.

◆ configure()

void configure ( )
virtual

Data manager virtual task configure function.

The configure function overloads the configure function of the task base class. It retrieves the configuration related to sensors by call of the function getCfgSensor to get the sensor threshold values, retrieve the configuration version by call of function getCfgGeneral, retrieve the configuration related to datalog by call of function getCfgDatalog and retrieve the configuration related to metrics by call of function getCfgMetrics. Configuration version, metrics interval, thresholds and activation flags will be stored in internal string variables of the MetricsBuilder class by calling its related setter functions.
Private variable bDispModeOff will be set according to configuration string dispmodeoff and private variable rms_z_thres according to rms_thres.

Send function to publish measurement JSON string to MQTT connector task.
