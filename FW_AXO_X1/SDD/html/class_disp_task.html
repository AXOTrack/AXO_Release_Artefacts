Detailed Description

Disp class derived from task base class.

The displacement task class implements the calulation of the displacement values based on the recorded acceleration data in z-direction. The calculation will be performed in the following steps:

  • Zero phase highpass filtering
  • Integration of filtered values to calculate velocity
  • Integration of velocity to calculate displacement
  • Removing linear trend from calculated displacement values (detrending) Zero phase highpass filtering:
    Zero phase highpass filtering is implemented according to matlab / python filtfilt function. The filter coefficients were taken over from python (see function butter_highpass in file disp_scipy.py in AXOTrack repository ).
    Integration of filtered values to calculate velocity:
    Calculation according to python disp_scipy.py
    Integration of velocity to calculate displacement:
    Calculation according to python disp_scipy.py
    Removing linear trend:
    Calculation according to python function signal.detrend().

◆ task()

void task ( void  )
virtual

DispTask task function.

The function directly enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eCalcDisplacementReq: Member function HandleCalcDisplacementReq will be called with message pointer pElem as parameter.
  • eRawDataStatus: Member function HandleRawDataStatus will be called with message pointer pElem as parameter.
  • eDispFirstEntries: Member function HandleDispFirstEntries will be called with message pointer pElem as parameter.
  • eDispLastEntries: Member function HandleDispLastEntries will be called with message pointer pElem as parameter.
  • eRawData: Member function HandleRawData will be called with message pointer pElem as parameter.
  • eStoredFwFiltVals: Member function InitBwFiltering will be called.
  • eFwFiltData: Member function HandleFwFiltData will be called with message pointer pElem as parameter.
  • eStoredBwFiltVals: Member function InitDispCalc will be called.
  • eBwFiltData: Member function HandleBwFiltData will be called with message pointer pElem as parameter.
  • eStoredSResultVals: Member function InitDetrend will be called.
  • eSResultData: Member function HandleSResultData will be called with message pointer pElem as parameter.
  • eStoredDResultVals: Flag bIdle will be set to true, to indicate that displacement calculation has been finished and message eSendDispDataReq will be send to HTTP Connector to start transmission of displacement data to HTTP server.

◆ HandleDispFirstEntries()

void HandleDispFirstEntries ( PoolElem *  pElem)
private

Function handle incoming raw data message for left initial condition calculation.

Parameters
pElemThe incoming message with the indication

The function extracts a structure of type adxl372_xyz_accel_data from the message payload and initialize a vector Z of type double with length nfact + 2 and the initial values 0. Than the function enters a loop and convert the received z raw values to double and copy them to the vector Z. With call of function prepare_leftpad provided by class CFilter the left initial conditions will be calculated and stored in vector leftpad. By calling function fwfilter_init the forward filtering will be prepaired and with call of function filterFirstIter also provided by class CFilter the first filter iteration will be initiated with the values in vector leftpad. Afterwards a structure of type AdxlRawDataProgress will be filled with start read position, the number of currently received entries, which in this case is totalNumEntries - (nfact + 1) to get the last recorded samples. Additionally the number of requested entries will be assigned, which is nfact + 1. Finally a message of type eDispLastEntries is send to ADXL store task to retrieve the last raw data values, which than will be used to calculate right initial conditions.

◆ HandleRawDataStatus()

void HandleRawDataStatus ( PoolElem *  pElem)
private

Function handle incoming raw data status indication message.

Parameters
pElemThe incoming message with the indication

The function extracts a structure of type AdxlRawDataStatus from the message payload and assign start read position and number of samples within structure to the member variables startReadPos and totalNumEntries. Member variable rcvdEntries will be initialized with 0. Please note, that start read position allways points to the beginning of the complete data area, not to the current read position. In case number of samples is greater than zero, the function retrieves the filter order and the length of edge transients, provided by class CFilter and assign it to the member variables nfact and nfilt. Afterward a structure of type AdxlRawDataProgress will be filled with start read position, the number of currently received entries, which in this case is 0 and with the number of requested entries, which is nfact + 2. A message of type eDispFirstEntries is than send to ADXL store task to retrieve the first raw data values, which than will be used to calculate left initial conditions. If number of samples is not greater than 0, the flag bIdle is set to true and the function returns.

FVector of type double with the filtered result values

The function initialize the remaining entries to write in variable remainingWrEntries and set variable requestedWrEntries to the maximum number of requested entries MAX_NUM_REQ_ENTRIES. In case remainingWrEntries are less than MAX_NUM_REQ_ENTRIES variable requestedWrEntries will be set to remainingWrEntries. Afterwards start write address will be assigned in structure progress of type AdxlFilterDataProgress in dependency of for- or backward filter results indicated by parameter mType. Start write address will be than readjusted. Please note, that the result vector will be stored in reverse order to be in correct direction for the next calculation step (for backward filtering or correct order after backward filtering). Structure progress will be filled with number of still sent entries and number of requested write entries. Afterwards the values in vector S will be stored in a temporary array in reverse order and copied in structure progress. By calling function sendMsg provided by class Mbox a message of type mType will be send to ADXL store task with structure progress in the payload. Finally member variable sentEntries will be increased by requestedWrEntries.

)
private

Function to calculate displacement before detrending.

Parameters
inIncoming vector of type double with for- and backward filtered values
VoutOutput vector with the integrated velocity values
SoutOutput vector with the integrated displacement values before detrending

The function initialize the current data index idx with difference of received entries minus requested entries. To avoid the influence of transient response of the filtering the first and last NUM_FILT_REMOVE entries will be deleted by setting them to 0. By checking the current data index the output values in range [0, NUM_FILT_REMOVE] and [totalNumEntries - NUM_FILT_REMOVE - 1, totalNumEntries -1] will be set to 0. Outside of these ranges the output values of the velocity Vout will be calculated by integration of the filtered input values and the displacement values before detrending Sout will be calculated by integration of the velocity.
