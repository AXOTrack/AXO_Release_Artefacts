SPI bus driver class implemented according to C++ singleton design pattern.

Detailed Description
The SPI bus class handles the initialization of the attached SPI slave and the communication over the SPI bus. Major part of configuration and initialization is located in the device tree, other parts are located in the init function of this class. storage of the acceleration raw See details in Device Driver Model documentation and in API documentation
The spibus class provides the init method.

◆ init()

void init ( )

The init function.

In case initialization still has been performed, indicated by flag init_done the function returns immediately, otherwise chip select gpios will be assigned by call of macro GPIO_DT_SPEC_GET and delay set to 0. The SPI controller device structure will be retrieved by calling macro DEVICE_DT_GET and function checks if the device is ready for usage with call of function device_is_ready. The return value of this function will be assigned to flag device_good. In case of an error a debug message at error level will be printed. Finally operation mode is set to loop mode, clock frequency is set to 8 MHz and chip select configuration is assigned before init_done flag is set to true.

Note
Adjustment of clock frequency for FRAM SPI is done here with line spi_cfg.frequency = 8000000U; The value must be unsigned integer in Hz.
