LTE monitor class derived from task base class.

Detailed Description
Todo:
RMS task receives the acceleration raw data from ADXL data store task, calculate RMS and with the help of class HitDetector hit values. RMS, medium and critical hit values will be send to RMS store task together with a timestamp. The class provides an idle detection timer, and for medium and critical hit detection an instance of class HitDetector. It provides the function update_rms to calculate RMS and hit values and to send them to the RMS store task.

    • RMS task (self)
    • ADXL data store task
  • Transmit

◆ task()

void task ( void  )
overridevirtual

MQTT connector task function.

Initially the function initialize the modem information library by calling function modem_info_init provided by the modem information library. The one shot IMSI / IMEI read timer will be started and firmware version will be retrieved by call of function getFwVersion provided from version module.
Afterwards the function enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On timer expiry the pointer pElem will be set to 0, to prevent releasing the message because it is allocated static. Afterwards the function read and print out IMSI and IMEI by calling the member functions getIMSI and getIMEI. Afterwards
  • eLteLinkQuality: On receiving the link quality from AT handler callback function at_ecsq_handler, the link quality value will be extracted from message payload and stored in private member variable linkQuality.
  • eLteReport: On receiving LTE report request from data manger task, LTE modem specific data will be read by calling the functions psm_read, getIMSI, getBattVoltage and getTemperature. These informations will be reported to data manger task by call of the functions reportLinkQuality, reportIMSI, reportPsmAct, reportBattVoltage and reportTemperature.
    Note
    that the sequence of the reports matters, since the receiver deduces "metrics complete" when modem voltage and temperature are received

Implements Task.

◆ psm_read()

void psm_read ( void  )

Function to retrieve power saving mode informations of the modem.

The method send the AT command "Power saving mode setting" +CPSMS to the modem and read the formatted response into the arguments psm_enabled, request_periodic_tau and request_active_time by calling function nrf_modem_at_scanf provided by the modem library of nrfxlib provided by Nordic semiconductor. In case of an error the method returns immediately. See Command Reference Guide, page 88ff for details. Afterwards function lte_lc_psm_get is called for getting the current PSM (Power Saving Mode) configurations for periodic TAU (Tracking Area Update) and active time, both in units of seconds. In case of an error the method returns immediately.
