#include <config.h>

Detailed Description

Configuration class implemented according to C++ singleton design pattern.

Detailed Description
The Config class handle the dynamic configuration of the AXO device basecd on a JSON configuration file located at the Jovia server. For each AXO device one configuration file is present on the server. On changing the content of the configuration file the server send the configuration via MQTT topic /device/{device_id}/config/update to the device if it is connected via LTE. On receiving the configuration, the device check if the configuration is valid and in this case store the new configuration in file system under /lfs/config.json and perform a reset.
On startup the device read the configuration file and provide the configuration values to the related modules in defined configuration structures. In case no configuration file is available in file system, the device store a default configuration before reading.
The config class uses the cJSON library, which is part of the Zephyr delivery.
The class provides methods for initialization of the class instance, to read the configuration file from file system, to parse the configuration JSON string into configuration structures, to print the JSON string, to retrieve group and key existence and the number of group entries and to get a specific configuration value. Furthermore it provides getter functions to retrieve every singe configuration structure.

void init ( )

Initialization function of the Config class.

The method define the default configuration as string, replaces the dummy entries in the default configuration, load the JSON configuration file from the file system and extract the the JSON string and store it in member variable input by calling member function loadFromFile. Afterwards the member function parse is called to retrieve a JSON root object from input string and store it in variable load_obj. Than the function checks if the loaded configuration is compatible to the version and structure of the default configuration by call of function isVersionValid. If compatible member function parse is called again to retrieve a JSON root object from input string and store it in member variable root_obj for further processing. Otherwise the default configuration string will be parssed in member variable root_obj and all existing configuration values from loaded configuration will be taken over from loaded configuration in variable load_obj in default configuration in member variable root_obj by call of member function mergeConfig and the default configuration than will be stored in file system by call of function saveToFile. Finally, by call of the member function parseAllValues the configuration values will be
extracted and stored into the provided configuration structures.

◆ parseAllValues()

void parseAllValues ( )

Function copy all configuration values in the provided configuration structures.

The method extract all configuration values from all groups of the JSON object by calling member getValue function for every single value. It copies the extracted values in the provided configuration structures of class Config. Name and version of the general group will be checked against the expected values. In case of inconsistencies a debug message at error level will be printed. If displacement calculation and displacement data transfer are disabled, the configuration is corrected to displacement data transfer disabled.

◆ Autocorrection()

void Autocorrection ( )

Function to automatically correct invalid configuration values.

By calling the member function getValue the function retrieves the activation and deactivation threshold strings and convert them to integer. Afterwards the function checks if:

  • Activation threshold is lower than the minimum allowed value. In this case activation threshold is set to the minimum allowed value
  • Activation threshold is higher than the maximum allowed value. In this case activation threshold is set to the maximum allowed value
  • Deactivation threshold is higher than the maximum allowed value. In this case deactivation threshold is set to the maximum allowed value
  • Deactivation threshold is lower than activation threshold and minimum allowed difference. In this case the deactivation threshold is set to the activation threshold plus the minimum allowed difference Afterwards the function retrieves the flags dispmodeoff and disptransferon by calling the member function getValue and checks if displacement calculation is disabled (dispmodeoff = true") and displacement value transfer is enabled (disptransferon = true). In this case disptransferon will be set to false.
