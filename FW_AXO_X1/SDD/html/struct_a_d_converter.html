Detailed Description

AD converter driver class derived from task base class.

Detailed Description
The AD converter driver class handle the reception of the analog values, convert them into digital values and report them to the registered listener task on request. For testing purposes cyclic timer handling is implemented and can be used for automatic cyclic sampling. The timers are organized in a vector and will be allocated in the autoSample method, which is only called by inactive testing scenarios.
The AD converter class provides methods to sample the data of the provided AD channels, to initialize the AD devices and to report the data to the registered listener tasks. It uses the public zephyr API for the sensor driver. See details in Zephyr ADC API documentation and for details of the ADC channel configuration in the devicetree Message Interaction:
  • Receive
    • Timer interrupt callback (only for testing purposes)
    • AD converter task (self, only for testing purposes)
    • Battery manager task
  • Transmit
    • Battery manager task
    • AD converter task (self, transmission in timer callback, only for testing purposes)

◆ task()

void task ( void  )
virtual

ADConverter task function.

Initially the AD converters will be initialized by calling member function initDevice. Then the timers for cyclic automatic sampling are started. Note, that these timer are only created for testing purposes. In productive software the vector of timers is empty. I2cBus instance. sensor device will be initialized by call of member function initDevice. For testing purposes a cyclic timer can be activated by commenting in the start method of the timer to initiate a cyclic sampling of the sensor data.
Afterwards the function enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On sample timer expiry sensor data will be sampled and reported to registered listeners by calling function sample. The pointer pElem will be set to 0, to prevent releasing the message because it is allocated static.
  • eAdcReport: On ADC report request sensor data will be sampled and reported to registered listeners by calling function sample.

Mbox *  listener 
)

Function to start cyclic automatic sampling if specified ADC channels.

Parameters
chmaskBitmask specifying which channels shall be reported to listener
intervalThe sample intervall in millliseconds
listenerAn optional listener, can be NULL if not required

The function allocates a structure of type AdcTimerInfo and assign the this pointer of the class instance and the specified channel bitmask to it. Afterwards it allocates a cyclic timer with duration and period specified by parameter interval and copy the pointer to the timer in vector timers by calling the vector class method push_back. Finally it checks if spezified mailbox pointer listener is not NULL and in this case it registers the listener for the channels specified in channel bitmask chmask by calling member function registerListener.

Note
This function is only implemented for testing purposes, in productive software this function is not called.

◆ sample()

void sample ( uint8_t  chmask)
private

Function to sample ADC data for specified channels.

Parameters
chmaskThe channel bitmask, specifying the channels which shall be sampled

The function allocates a structure of type struct adc_sequence which defines an ADC sampling sequence with an 16 bit unsigned integer type. Than it enters a loop over the ADC channels and check if bit at position i is set in channel bitmask chmask. If bit is set, it initialize the struct sequence of type adc_sequence with the information provided array adc_channels of type struct adc_dt_spec at index i by calling function adc_sequence_init_dt provided by Zephyr ADC API. With call of function adc_read also provided by Zephyr ADC API it reads the raw value into structure sequence. Afterwards the raw value will be assigned to to local varible val_mv and converted to millivolt by call of function adc_raw_to_millivolts_dt also provided by Zephyr ADC API, which uses the ADC specification from device tree from structure struct adc_dt_spec provided by array adc_channels at position i. Finally the listeners will be informed by calling member function notifyListener.
