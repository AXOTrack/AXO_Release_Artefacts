Detailed Description

MQTT connector class derived from task base class.

Detailed Description
MQTT connector task handle the connection to the MQTT broker. It receives messages from the broker via static callback member functions, which needs to be registered in MQTT helper layer provided by the nRF IoT library.
See details of nRF library
See details of MQTT Client library API The MqttConnector class provides five handler functions to receive MQTT broker events, methods to request connection, disconnection and subsciption, to abort connection to broker and to publish topics to the broker. Furthermore it provides a method to read the hardware device ID (modem's IMEI). Message Interaction:
  • Receive
    • External PDN and LTE Link events
    • Timer interrupt callback
    • MQTT connector task (self)
    • Firmware update task
    • Firmware update control task
    • LTE connector task
    • Data manager task
  • Transmit
    • MQTT connector task (self)
    • Firmware update control task
    • MQTT connector task

◆ task()

void task ( void  )
virtual

MQTT connector task function.

Initially the function register the callback functions for MQTT broker events by assigning them in structure cfg of type mqtt_helper_cfg and calling the function mqtt_helper_init provided by the nRF IoT library.
Afterwards the function enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On timer expiry the function checks if a LTE connection is established and MQTT connection was lost. In this case the member function connect will be called to send a new connect request to the MQTT broker. Afterwards the pointer pElem will be set to 0, to prevent releasing the message because it is allocated static.
  • eCmdAccept: If MQTT broker is connected, a accepted response will be send to the MQTT broker by call of the private member function PublishAcceptedResponse.
  • eCmdReject: If MQTT broker is connected, a rejected response will be send to the MQTT broker by call of the private member function PublishRejectedResponse.
  • eOtaStart: If MQTT broker is connected, a accepted response will be send to the MQTT broker by call of the private member function PublishAcceptedResponse. Finally a subscribe request will be send to MQTT broker by calling the member function ota_subscribe.
  • eFwAccept: If MQTT broker is connected, a accepted response will be send to the MQTT broker by call of the private member function PublishAcceptedResponse.
  • eFwReject: If MQTT broker is connected, a rejected response will be send to the MQTT broker by call of the private member function PublishRejectedResponse.
  • eFwDlSuccess: If MQTT broker is connected, a accepted response will be send to the MQTT broker by call of the private member function PublishDlResult.
  • eFwDlFailed: If MQTT broker is connected, a rejected response will be send to the MQTT broker by call of the private member function PublishDlResult.
  • eMqttConnected: Message will be forwarded to power manager task, the pointer pElem will be set to 0, to prevent releasing the message. The flag connected will be set to true and the reconnect timer will be stopped by calling stop function of the timer class. Afterwards the function checks if public TLS key is already send to backend. In this case a debug message at warning level will be printed, otherwise the key transfer to backend will be iniatialized y sending message eKeyTransferReq to its own (MQTT connectors) mailbox.
  • eMqttDisconnected: The flag connected will be set to false, to prevent inconsistent library states the member function abort will be called and in cast LTE is connected the reconnect timer will be started by calling start function of the timer class.
  • eLteSleepEnter: Will be ignored, only debug message at info level will be printed.
  • eLteSleepExit: Will be ignored, only debug message at info level will be printed.
  • eKeyTransferReq: If MQTT broker is connected, the public key will be retrieved by call of function getPublicKey provided by class KeyGen and the key will be send to MQTT broken in pubkey topic by calling private member function PublishData. Finally by key transfered indicator will be set by call of function setKeyTransfered, also provided by class KeyGen
  • eMetrics: If MQTT broker is connected, the message payload will be copied to topic payload structure and send to MQTT broker in metrics topic by call of private member function PublishData
  • eMeasurements: If MQTT broker is connected, the message payload will be copied to topic payload structure and send to MQTT broker in measurement topic by call of private member function PublishData
  • eLteConnected: If LTE is not connected, the flag lte_connected will be set to true and if MQTT broker is not connected, the member function connect will be called to send a new connect request to the MQTT broker.
  • eLteDisconnected: The flag lte_connected will be set to false, the reconnect timer will be stopped by calling stop function of the timer class and the member function disconnect will be called to send a disconnect request to the MQTT broker.
  • eJsonConfig: Call of member function HandleNewConfiguration.
  • eFactoryDefault: Call of member function HandleFactoryDefault.
  • eColdStartReq: Call of member function HandleColdStartReq.
  • eBoostOnReq: Call of member function HandleBoostOnReq.
  • eBoostOffReq: Call of member function HandleBoostOffReq.
  • eFsErase: Call of member function HandleFsErase
  • eFsStatus: Call of member function HandleFsStatus
  • eDisableADXL: Call of member function HandleDisableADXL
  • eEnableADXL: Call of member function HandleEnableADXL
  • eGetConfig: Call of member function HandleGetConfig

struct mqtt_helper_buf  payload 
)
static

Callback handlers from MQTT helper library for a subscribed topic from broker.

Parameters
topicThe topic MQTT connector is subscribed
payloadThe payload of the topic

The function is called when MQTT connection has received a subscribed topic from the broker. Topic and payload will be casted to strings and the command in payload string will be identified with string find method.
In case command is OTA_START from topic /device/{device_id}/command and a firmware update is requested the function request a message from message pool, set message type to eOtaStart, set eMqttConnector as sender and send message to firmware update control task's mailbox.
In case the string fw_version is found in the payload of topic /device/{device_id}/ota/update the function also request a message from message pool, set message type to eFwInfo, set eMqttConnector as sender and send message to firmware update control task's mailbox.
In case command is RESET a reset will be performed by calling function performSwReset provided by class ResetCtrl.
In case command is CFG_FACTORY_DEFAULT a message of type eFactoryDefault will be send to the MQTT Connector mailbox. In case command is COLDSTART a message of type eColdStartReq will be send to the MQTT Connector mailbox. In case command is BOOST_ON a message of type eBoostOnReq will be send to the MQTT Connector mailbox. In case command is BOOST_OFF a message of type eBoostOffReq will be send to the MQTT Connector mailbox. In case command is FS_ERASE a message of type eFsErase will be send to the MQTT Connector mailbox. In case command is FS_STAT a message of type eFsStatus will be send to the MQTT Connector mailbox. In case command is DISABLE_ADXL a message of type eDisableADXL will be send to the MQTT Connector mailbox. In case command is ENABLE_ADXL a message of type eEnableADXL will be send to the MQTT Connector mailbox. In case command is GET_CONFIG a message of type eGetConfig will be send to the MQTT Connector mailbox. In case command is SEND_PUBLIC_KEY a message of type eKeyTransferReq will be send to the MQTT Connector mailbox.

void HandleNewConfiguration ( PoolElem *  pElem)
private

Handler function for new incoming JSON configuration.

Parameters
pElemPointer to the received message

The function copy the message payload of pElem to topic payload structure, initialize the topic with the config topic and extract the JSON configuration string in variable rx. Afterwards the function checks if the configuration is valid by calling method isValid provided by the Config class. If configuration is valid, the topic will be enhanced with string "/accepted" and published by call of the member function publish. Afterwards configuration will be stored in file system by calling function saveToFile also provided by Config class, before a reset is performed by calling calling function performSwReset provided by class ResetCtrl. If configuration is invalid, the topic will be enhanced with string "/rejected" and published by call of the member function publish.
