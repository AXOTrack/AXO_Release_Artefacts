Detailed Description
This file defines the stack sizes for each tasks of the AXO Track software and allocate each task statically. The main function creates and starts the peviously allocated tasks before entering the endless loop.

int main ( )

main function of AXO Track firmware

The main function load dynamic configuration from file and initialize the config variables by call of function init provided by Config class, it call function init provided by KeyGen class to generate and store private and public TLS keys if not still done. Afterwards it creates and starts the static allocated tasks in following order:

  • create and start BoostConverter task (boost)
  • create and start PowerMgr task (pwrmgr)
  • create LteConnector task (lte)
    • register mqtt and power manager instances to listen to selected LTE events
  • create LteMonitor task (ltemon)
    • register notifications of link quality and network
  • create, configure and start MqttConnector task (mqtt)
  • create Shtcx (temperature + humidity sensor) task (shtcx)
  • create and start ADC task (adc)
  • create, configure and start battery manager task (battery)
  • create, configure and start data manager task (dmgr)
  • start LteMonitor task (ltemon)
  • create and start accelerometer task adxlrms
  • create, configure and start accelerometer task adxlstore
  • create and start accelerometer task rmsStore
  • create and start accelerometer task adxlsample
  • create and start firmware updata task (fwupd)
  • create and start firmware updata control task (fwctrl)
  • create and start reset control task (resetctrl)
  • start LteConnector task (lte)
  • call function displayFwVersion
  • disable UART (for release builds only)
  • enter endless loop
Returns
The main function never returns, defined return value is 0.
