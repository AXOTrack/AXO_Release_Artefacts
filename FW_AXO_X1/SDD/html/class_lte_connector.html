LTE connector class derived from task base class.

Detailed Description
LTE connector task handle the connection to the LTE modem, modem wake up and sleep behaviour and inform its listener tasks (power manager and MQTT connector) about these events. It uses the nRF PDN library API and LTE Link API via AT commands to communicate with LTE modem and to receive PDN and LTE Link events. See details in LTE Link API documentation and for PDN library API in PDN library API documentation
The LteConnector class provides two handler functions to receive LTE link and PDN events, a method to send messages to its own (LTE connector's) mailbox, which is used by the handler functions to transform external events to internal messages of type PoolElem. It provide a timer callback function and a timer expired function, used to transform external timer expiry event from interrupt context to internal timer expired message of type PoolElem. Furthermore it provide methods to register listener tasks, to notify listener tasks about received external events and to report eDRX values to data manager task. Message Interaction:
  • Receive
    • External PDN and LTE Link events
    • Timer interrupt callback
    • LTE connector task (self)
    • Power manager task
  • Transmit
    • LTE connector task (self)
    • Power manager task
    • MQTT connector task
    • Data manager task

◆ task()

void task ( void  )
overridevirtual

LTE connector task function.

Initially the function register the callback functions for PDN events by calling the function pdn_default_ctx_cb_reg from nRF PDN library API and for LTE events by calling function lte_lc_register_handler from nRF public API for the LTE Link Controller. With call of function lte_lc_modem_events_enable the LTE connector enable the modem domain events, which will be handled by LTE link event handler lte_event_handler. LTE module will be initialized and modem will be configured by call of function lte_lc_init and modem's functional mode is set to LTE_LC_FUNC_MODE_ACTIVATE_UICC by calling function lte_lc_func_mode_set, which activate UICC (Universal Integrated Circuit Card) notifications and is required to read IMSI without network connection.
Afterwards the function enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On timer expiry the pointer pElem will be set to 0, to prevent releasing the message because it is allocated static. Afterwards the function checks if flag bConnectRetry is true and in this case it triggers a reset by call of function sys_reboot. If flag bConnectRetry is not true, the function checks if LTE is still connected (flag bLteConnected is false). In this case the function switch of LTE modem by calling the functions lte_lc_offline, lte_lc_deinit and lte_lc_power_off. Than timer will be sterted with duration of connectRetryTime (24 hours) and flag bConnectRetry is set to true.
  • eLteConnected: Listeners will be informed by calling member function notify with message of type eLteConnected and flag bLteConnected will be set to true.
  • eLteDisconnected: Listeners will be informed by calling member function notify with message of type eLteDisconnected and flag bLteConnected will be set to false.
  • eLteSleepEnter: Listeners will be informed by calling member function notify with message of type eLteSleepEnter.
  • eLteSleepExit: Listeners will be informed by calling member function notify with message of type eLteSleepExit and a timer will be started with duration connectTimeout (15 min).
  • eIp6Up: Will be ignored, only debug message at info level will be printed.
  • eIp6Down: Will be ignored, only debug message at info level will be printed.
  • eResume: With call of function lte_lc_normal modem will be switched to full functionality. Power Saving Mode will be enabled enabled by calling function lte_lc_psm_req and a LTE connection will be established by calling function lte_lc_connect_async.
  • eSuspend: Will be ignored, only debug message at info level will be printed

◆ lte_event_handler()

void lte_event_handler ( const struct lte_lc_evt *const  evt)
static

LteConnector LTE link event handler function.

Parameters
evtPointer to the received LTE link event of type struct lte_lc_evt

The handler function notifies the application of sleep, wakeup and DRX events from LTE modem. These events will be mapped to internal message types and send to LTE connectors own or data manager mailbox before function returns.
Mapping is as follows:

  • LTE_LC_EVT_MODEM_SLEEP_ENTER -> eLteSleepEnter to own mailbox
  • LTE_LC_EVT_MODEM_SLEEP_EXIT -> eLteSleepExit to own mailbox
  • all other LTE link events: will be ignored (only debug message) In case of unexpected events the function print an error debug message returns immediatly.

LteConnector PDN event handler function.

The handler function notifies the application of events related to the default PDN context, CID 0. The PDN events will be mapped to internal message types and send to LTE connectors own mailbox before function returns. Mapping is as follows:

  • PDN_EVENT_CNEC_ESM: will be ignored (only debug message)
  • PDN_EVENT_ACTIVATED -> eLteConnected
  • PDN_EVENT_DEACTIVATED -> eLteDisconnected
  • PDN_EVENT_IPV6_UP -> eIp6Up
  • PDN_EVENT_IPV6_DOWN -> eIp6Down In case of unexpected events the function print an error debug message returns immediatly
