#include <keygen.h>

Detailed Description

TLS key generation class implemented according to C++ singleton design pattern.

The KeyGen class use the mbedtls crypto library. See details in Mbed TLS API documentation It provide a init method which generate the TLS key pair if not exisiting,
a method to check if keys are generated, a method to check if public key is transfered to backend, methods to retrieve public and private key, methods to priont out a key, a method to set the indicator that public key is transfered and a methos to delete all key related files. This last function is only for development purposes.

◆ init()

void init ( )

The init function.

The function checks if both TLS key files still exist. If yes the member variable keygen_done will be set to true. Aftwards the function checks if the "key tranferred" file exist and in this case it set member variable transfer_done. If key generation is still done, indicated by flag keygen_done the function returns immediately.
If key generation is not done, the key generation will be initialized by call of the function mbedtls_ctr_drbg_init provided by mbedtls crypto library, which initializes the CTR_DRBG context and prepare it for seed generation. Afterwards the entropy controller device structure will be retrieved from device tree by calling macro DEVICE_DT_GET and the function checks if the device is ready for usage with call of function device_is_ready. If device is not ready a debug message at error level will be printed, otherwise seed will be generated by call of function mbedtls_ctr_drbg_seed, which also set up the CTR_DRBG entropy source for future reseeds.
With call of the functions mbedtls_pk_init and mbedtls_pk_setup provided by the mbedtls crypto library the mbedTLS structure of type mbedtls_pk_context will be initialized. Calling function mbedtls_ecp_gen_key generates the TLS keypair. Calling function mbedtls_pk_write_key_pem extract the private key into a string, which will than be stored into file system. Calling function mbedtls_pk_write_key_pem extract the public key into a string, which will than also be stored into file system. Finally the called functions mbedtls_pk_free and mbedtls_ctr_drbg_free release the components of the context variables.
