Detailed Description
Mailboxes work like FIFO buffers for messages. Every task allocates one object of class Mbox for receiving messages. Internally Zephyr's kernel message queue service is used for implementation (see mqueue). Every task task receives only messages on their own mailbox. Tasks can send/write messages to any mailbox in the system.
Tasks waiting for messages, they are blocking on the read() function if the mailbox is empty. If a message is in the mailbox the task wakes up, read and process the message. Sending a message means writing to the mailbox of the receiver's task. A sending task blocks on the write() function in case the mailbox is full. This behaviour may be modified by adding a timeout on read/write calls.
For sending a message, the target mailbox must be known. The Mbox class implements a registry and lookup service for all Mbox objects created. All tasks register their mailboxes together with a their unique task ID. Via the task ID, a pointer to the associated mailbox can be retrieved and used for sending/writing a message.
