Detailed Description

Boost converter class derived from task base class.

Detailed Description
The boost converter class switches boost converter on and off. It provides methods to set GPIO pin P0.21 in active and inactive state to switch boost converter on and off and a timer to control the duration of boost controller activity. Message Interaction:
  • Receive
    • Timer interrupt callback
    • Boost converter task (self)
    • Power manager task
  • Transmit
    • Boost converter task (self)

BoostConverter ( const char *  name = "??")
explicit

Boost converter task constructor.

Parameters
nameThe name of the class

The constructor allocate and register a mailbox and allocate the timer to supervise the maximum activation time period. By calling macro GPIO_DT_SPEC_GET it retrieves the GPIO pin information structure gpio from devicetree and checks with call of function device_is_ready if the gpio is ready for use. The private flag device_good, indicating that hardware initialization was successful, will be set according to the return value of function device_is_ready. If initialization was not successful an error message will be printed.
Flag isEnabled will be set to false, in case file BOOST_ENABLED_FILENAME exists, flag will be set to true.

◆ task()

void task ( void  )
virtual

Boost converter task function.

The function enters the endless loop and wait for messages. Task wakes up when a message is in the mailbox and read the message by calling mailbox read method which returns the pointer pElem to the message. All expected messages will be handled, in case of an unexpected message an error debug message will be triggered. Finally the message will be released by calling class PoolElem method free, if the pointer pElem is not explicitly set to 0.

Message Handling:

  • eTimerExpired: On timer expiry the pointer pElem will be set to 0, to prevent releasing the message because it is allocated static. Finally boost converter is switched off by calling function off().
  • eResume: On reciving the resume message from power manager the timeout value for the maximum activation time period will be extracted from message payload and the timer will be started with this timeout value. Afterwards boost converter is switched on by calling function on().
  • eSuspend: On reciving the suspend message from power manager, timer will be stopped and boost converter is switched off by calling function off().
  • eBoostOnReq: Boost converter will be enabled by setting flag isEnabled to true and file BOOST_ENABLED_FILENAME will be written to file system as persistent indication.
  • eBoostOffReq: Boost converter will be enabled by setting flag isEnabled to false and file BOOST_ENABLED_FILENAME will be deleted.
