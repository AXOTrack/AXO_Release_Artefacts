11 * \anchor mbox
12 *
13 * \description Mailboxes work like FIFO buffers for messages. Every task
14 * allocates one object of class Mbox for receiving messages. Internally
15 * Zephyr's kernel message queue service is used for implementation (see \ref
16 * mqueue). Every task task receives only messages on their own mailbox. Tasks
17 * can send/write messages to any mailbox in the system. \n Tasks waiting for
18 * messages, they are blocking on the read() function if the mailbox is empty.
19 * If a message is in the mailbox the task wakes up, read and process the
20 * message. Sending a message means writing to the mailbox of the receiver's
21 * task. A sending task blocks on the write() function in case the mailbox is
22 * full. This behaviour may be modified by adding a timeout on read/write calls.
23 * \n For sending a message, the target mailbox must be known. The Mbox class
24 * implements a registry and lookup service for all Mbox objects created. All
25 * tasks register their mailboxes together with a their unique task ID. Via the
26 * task ID, a pointer to the associated mailbox can be retrieved and used for
27 * sending/writing a message.
